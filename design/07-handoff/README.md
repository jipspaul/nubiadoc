# Handoff dev

> Specs de passation vers le dev **Flutter (Bloc)**, à générer avec la skill **`design-handoff`**.

Pour chaque écran/composant : layout & espacements, tokens utilisés, props/variantes, états d'interaction, breakpoints responsive (mobile patient vs desktop back-office), cas limites, animations.

Cible d'implémentation : widgets Flutter + thème issu du `03-design-system/`. Relier chaque écran à son épic (`E3.x` / `E4.x` / `E5.x`) et à l'issue `NUB-T*` correspondante (`../../docs/09`).
